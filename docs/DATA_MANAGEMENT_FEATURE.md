## 概要

### 目的
CSVマスターデータをブラウザ上で編集・保存できる機能を追加し、エンドクライアントが提出したデータを今後更新・保守しやすくする。

### 期待される効果
- **保守性の向上**: CSVデータの更新をコード変更なしで行える
- **運用の柔軟性**: データの調整や検証を容易にする
- **透明性の向上**: データの内容を確認・編集できるUIを提供
- **エラー防止**: バリデーションにより不正なデータ入力を防止

### 1. monthly-index.csv（月次変動係数）

**シミュレーションへの影響**: 高（宿泊単価に直接乗算）

**構造**:
```csv
,1月,2月,3月,4月,5月,6月,7月,8月,9月,10月,11月,12月
北海道,0.88,0.93,0.83,0.68,0.87,1.04,1.27,1.34,1.16,1.06,0.85,1.04
青森県,0.78,0.77,0.72,0.60,0.77,0.92,1.13,1.19,1.03,0.94,0.75,0.92
...
```

**列定義**:
| 列 | 型 | 説明 | 制約 |
|----|-----|------|------|
| 第1列 | string | 都道府県名 | 47都道府県 |
| 1月〜12月 | number | 月次変動係数 | 0.5〜2.0 |

**計算式での使用**:
```
実効宿泊単価 = 基準単価 × 月次変動係数 × 築年数係数
```

**編集時の注意点**:
- 1.0が基準値（増減なし）
- 0.6〜1.78が現在の値の範囲
- 夏季（7-8月）は高め、閑散期（4月）は低めが一般的

### 2. working-days.csv / working-days-normal.csv（稼働日数）

**シミュレーションへの影響**: 高（売上計算の基礎）

**構造**:
```csv
,1月,2月,3月,4月,5月,6月,7月,8月,9月,10月,11月,12月
日数,31,28,31,30,31,30,31,31,30,31,30,31
北海道,10,9,9,7,10,11,14,15,13,12,9,12
青森県,9,8,8,6,9,10,13,14,12,11,8,11
...
```

**列定義**:
| 列 | 型 | 説明 | 制約 |
|----|-----|------|------|
| 第1列 | string | 都道府県名 | 47都道府県 + "日数"行 |
| 1月〜12月 | number | 稼働予測日数 | 1〜月の日数 |

**2ファイルの違い**:
- `working-days.csv`: 通常運用（年間制限なし）
- `working-days-normal.csv`: 民泊新法適用（年間180日制限）

**バリデーション**:
- 各月の値 ≤ その月の日数
- `working-days-normal.csv`は年間合計 ≤ 180日

### 3. base-price.csv（基本宿泊単価）

**シミュレーションへの影響**: 高（売上計算の基礎単価）

**構造**:
```csv
,エリア,物価係数,1singleBP,1,2ダブル,2ツイン,3,4,5,...,20
北海道,北海道,0.961,8050,8400,9650,10900,12950,15000,...
東京都,東京,1.047,9600,10050,11550,13050,15550,18050,...
```

**列定義**:
| 列 | 型 | 説明 | 制約 |
|----|-----|------|------|
| 第1列 | string | 都道府県名 | 47都道府県 |
| エリア | string | 広域圏分類 | 北海道/東北/関東/... |
| 物価係数 | number | 地域補正係数 | 0.9〜1.1 |
| 1singleBP | number | シングル単価 | 正の整数 |
| 1〜20 | number | 定員別単価 | 正の整数 |
| 2ダブル | number | 2名ダブル単価 | 正の整数 |
| 2ツイン | number | 2名ツイン単価 | 正の整数 |

**編集時の注意点**:
- 47行（都道府県）× 約25列の大規模テーブル
- 定員2名の場合、ベッド構成により「2ダブル」または「2ツイン」を参照

### 4. expenses.csv（固定費）

**シミュレーションへの影響**: 中（月額固定費に加算）

**構造**:
```csv
rooms_count,trash,internet
1,8000,5000
6,15000,10000
16,22000,15000
26,30000,20000
41,40000,30000
51,50000,50000
100,100000,80000
```

**列定義**:
| 列 | 型 | 説明 | 制約 |
|----|-----|------|------|
| rooms_count | number | 部屋数ティア閾値 | 1,6,16,26,41,51,100 |
| trash | number | ゴミ処理費（月額） | 正の整数 |
| internet | number | インターネット費（月額） | 正の整数 |

**ティア判定ロジック**:
```
部屋数1〜5 → rooms_count=1 の行を参照
部屋数6〜15 → rooms_count=6 の行を参照
...
```

### 5. building-age.csv（築年数係数）

**シミュレーションへの影響**: 中（宿泊単価に乗算）

**構造**:
```csv
ageRange,isRenewedFalse,isRenewedTrue
1,1.15,1.15
2,1.05,1.05
3,0.9,1.0
4,0.8,0.95
```

**列定義**:
| 列 | 型 | 説明 | 制約 |
|----|-----|------|------|
| ageRange | number | 築年数区分 | 1〜4 |
| isRenewedFalse | number | リノベなし係数 | 0.7〜1.2 |
| isRenewedTrue | number | リノベあり係数 | 0.7〜1.2 |

**築年数区分の意味**:
- 1: 新築〜4年
- 2: 5〜15年
- 3: 16〜40年
- 4: 41年以上

### 6. room-rank.csv（水道光熱費ランク）

**シミュレーションへの影響**: 中（月額経費に加算）

**構造**:
```csv
minArea,maxArea,rank,waterutilityCost,minCapacity,maxCapacity
1,25,A,11000,1,4
26,40,B,26000,1,5
41,65,C,28000,2,7
66,100,D,38000,3,29
```

**列定義**:
| 列 | 型 | 説明 | 制約 |
|----|-----|------|------|
| minArea | number | 最小面積（㎡） | 正の整数 |
| maxArea | number | 最大面積（㎡） | > minArea |
| rank | string | ランク | A/B/C/D |
| waterutilityCost | number | 水道光熱費（月額） | 正の整数 |
| minCapacity | number | 最小収容人数 | 正の整数 |
| maxCapacity | number | 最大収容人数 | ≥ minCapacity |

## API仕様

### GET /api/csv-management

**ファイル一覧取得**:
```
GET /api/csv-management?action=list
```

**レスポンス**:
```json
{
  "files": [
    {
      "name": "monthly-index.csv",
      "displayName": "月次変動係数",
      "description": "都道府県×月別の需要変動係数",
      "lastModified": "2026-01-16T10:00:00Z",
      "rowCount": 48,
      "priority": "high"
    },
    ...
  ]
}
```

**個別ファイル取得**:
```
GET /api/csv-management?file=monthly-index.csv
```

**レスポンス**:
```json
{
  "fileName": "monthly-index.csv",
  "content": "CSVテキスト...",
  "headers": ["", "1月", "2月", ...],
  "rows": [["北海道", "0.88", "0.93", ...], ...],
  "schema": {
    "columns": [
      { "name": "", "type": "string", "required": true },
      { "name": "1月", "type": "number", "min": 0.5, "max": 2.0 },
      ...
    ]
  }
}
```

### POST /api/csv-management

**ファイル保存**:
```
POST /api/csv-management
Content-Type: application/json

{
  "fileName": "monthly-index.csv",
  "content": "更新後のCSVテキスト",
  "createBackup": true
}
```

**レスポンス**:
```json
{
  "success": true,
  "message": "保存しました",
  "backupPath": "monthly-index.csv.backup.20260116-100000"
}
```

**エラーレスポンス**:
```json
{
  "success": false,
  "error": "バリデーションエラー",
  "details": [
    { "row": 3, "column": "7月", "message": "値が範囲外です（0.5〜2.0）" }
  ]
}
```

## バリデーションルール

### 共通ルール

| ルール | 説明 |
|--------|------|
| 必須チェック | 空セルは許可しない（特定列を除く） |
| 型チェック | 数値列は数値のみ許可 |
| 範囲チェック | 各列の定義された範囲内のみ許可 |
| 形式チェック | UTF-8エンコーディング必須 |

### ファイル別バリデーション

```typescript
const validationRules: Record<string, ValidationSchema> = {
  "monthly-index.csv": {
    columns: {
      "": { type: "string", required: true, enum: PREFECTURES },
      "1月": { type: "number", min: 0.5, max: 2.0 },
      "2月": { type: "number", min: 0.5, max: 2.0 },
      // ... 12月まで
    },
    rowCount: { exact: 48 }, // 47都道府県 + ヘッダー
  },

  "working-days.csv": {
    columns: {
      "": { type: "string", required: true },
      "1月": { type: "integer", min: 1, max: 31 },
      "2月": { type: "integer", min: 1, max: 29 },
      // ... 月ごとに最大日数が異なる
    },
    customValidation: (data) => {
      // working-days-normal.csv の場合は年間合計 ≤ 180
    },
  },

  "expenses.csv": {
    columns: {
      "rooms_count": { type: "integer", enum: [1, 6, 16, 26, 41, 51, 100] },
      "trash": { type: "integer", min: 0 },
      "internet": { type: "integer", min: 0 },
    },
    sortedBy: "rooms_count", // 昇順必須
  },

  "building-age.csv": {
    columns: {
      "ageRange": { type: "integer", enum: [1, 2, 3, 4] },
      "isRenewedFalse": { type: "number", min: 0.7, max: 1.2 },
      "isRenewedTrue": { type: "number", min: 0.7, max: 1.2 },
    },
    customValidation: (data) => {
      // isRenewedTrue >= isRenewedFalse を確認
    },
  },
};
```

### 編集インタラクション

1. **セル編集**: ダブルクリックまたはEnterで編集モード

3. **変更追跡**: 変更されたセルは黄色背景でハイライト
4. **キーボード**: Tab/Shift+Tabでセル移動、Escでキャンセル
